# payment-webhook-idempotency-system
A backend system demonstrating how to safely process payment provider webhooks using idempotency and event validation, preventing duplicate transactions caused by retries.

This project simulates real-world payment integrations where reliability and correctness are critical.

Key Features
Webhook endpoint simulation
Event storage and tracking
Idempotency key enforcement
Duplicate transaction prevention
Retry handling logic
Signature verification (mock or real)
